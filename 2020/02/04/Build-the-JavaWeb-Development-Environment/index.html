开发环境的安装及配置

JDK+JRE的安装及配置

  1. 进入 JDK 和 JRE 下载界面,点击下图红框的按钮，然后选择适合自己的系统安装包下载


  1. 解压下载好的压缩包，个人习惯将文件夹名字修改成JDK和JDR
系统 步骤
Windows 下载.exe文件，按步骤安装
或者下载.zip文件解压
Linux 1. 将安装包安装到/usr/local 路径下
2. 解压压缩包
 · 如果格式为.tar.gz后缀 使用命令$ tar -zxvf
 · 如果格式为.rpm后缀 使用命令$ rpm -ivh
Mac OS 下载.dmg 文件
按安装程序步骤安装
  • JDK的目录结构
    • bin目录: JDK开发工具的可执行文件。
    • lib目录 : 开发工具使用的归档包文件。
    • jre : Java 运行时环境的根目录，包含Java虚拟机，运行时的类包和Java应用启动器，但不包含开发环境中的开发工具。
    • demo : 含有源代码的程序示例。
    • include: 包含C语言头文件,支持Java本地接口与Java虚拟机调试程序接口的本地编程技术。
  1. 配置环境变量
系统 步骤
Windows 1. 打开控制面板
2. 点击高级系统设置
3. 点击 高级 里面的环境变量
4. 在系统变量下面点击 新建 ,新建JDK环境变量
 · 变量名 JAVA_HOME
 · 变量值 JDK的目录(例如C:\Program Files\Java\JDK)
5.在系统变量下面点击 新建 ,新建JRE环境变量
 · 变量名 JRE_HOME
 · 变量值 JRE的目录(例如C:\Program Files\Java\JRE)
6.双击系统变量下的Path变量
7. 新建字段：%JAVA_HOMR%\bin 和 %JRE_HOMR%\bin
Linux 1. 修改环境变量(前提会用 vi 编辑器)
 · 终端输入$ vi /etc/profile
 · 在文件最后添加
  export JAVA_HOME=/usr/local/java/jdk
  export CLASSPATH=.:$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
 export PATH=$PATH:$JAVA_HOME/bin
2. 保存并退出Vi
3. 终端输入$ source /etc/profile
Mac OS 1. 终端输入$ /usr/libexec/java_home -V 获取JDK路径
2. 新建一个.bash_profile的隐藏配置文件，打开终端，在终端中输入$ touch .bash_profile 命令：
3.打开.bash_profile文件，在终端中输入命令$ open -e .bash_profile
4. 添加字段
  JAVA_HOME=/Library/Java/JavaVirtual
  Machines/jdk-11.jdk/Contents/Home
 PATH=$JAVA_HOME/bin:$PATH:.
 CLASSPATH=$JAVA_HOME/lib/tools.jar:$JAVA_HOME/lib/dt.jar:.
 export JAVA_HOME
 export PATH
 export CLASSPATH
5.使配置文件生效，在终端中输入以下命令：$ source .bash_profile
  1. 系统测试
系统 测试
Windows 1.打开CMD(win + R 输入CMD)
2. 输入 java -version 回车
3.如果出现java version “1.8.0_241”
类似的语句，则安装成功
Linux 1.打开Terminal
2. 输入 java -version 回车
3.如果出现java version “1.8.0_241”
类似的语句，则安装成功
Mac OS 检查刚刚设置的环境变量的路径，查看是否配置成功。在终端中输入命令$ echo $JAVA_HOME，如果能正确查看java的版本信息，即表示已经配置成功。

Tomcat 的安装及配置

  1. 点击Tomcat进入版本列表里面，逐次进入目录里面的/bin目录如http://mirror.bit.edu.cn/apache/tomcat/tomcat-9/v9.0.30/bin/ 下载系统对应的版本

  2. 解压下载好的压缩包

  • 建议在安装时候，将文件夹命名为 Tomcat
系统 步骤
Windows 下载.exe文件，按步骤安装
或者下载.zip文件解压
Linux 1. 将安装包安装到/usr/local 路径下
2. 解压压缩包
 · 如果格式为.tar.gz后缀 使用命令$ tar -zxvf
 · 如果格式为.rpm后缀 使用命令$ rpm -ivh
Mac OS 下载tar.gz文件 ，解压后放入 /Library 路径下

Mac OS 寻找/Library 目录

  • 点击finder–>用户–>你电脑的名字–>资源库(有的也叫/Library)

  • 有些苹果将library目录隐藏起来了，打开Finder，按下shift+command+g，输入“~/Library”（输入引号里面的），再按回车就到了。

  1. 配置Tomcat环境
系统 步骤
Windows 1. 打开控制面板
2. 点击高级系统设置
3. 点击 高级 里面的环境变量
4. 在系统变量下面点击 新建 ,新建Tomcat环境变量
 · 变量名 CATALINA_HOME
 · 变量值 Tomcat的目录(例如C:\Program Files\Tomcat)
5.双击系统变量下的Path变量
7. 新建字段：%CATALINA_HOME%\bin 和 %CATALINA_HOME%\lib
Linux 修改环境变量(前提会用 vi 编辑器)
 · 终端输入$ vi /etc/profile
 · 在文件最后添加
  export CATALINA_HOME=/usr/local/Tomcat
  export PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin
 · 终端输入$ source /etc/profile
Mac OS 无需设施环境变量
  1. 系统测试
系统 测试
Windows 1.打开CMD(win + R 输入CMD)
2. 输入startup 回车
3.打开浏览器，在地址栏中输入localhost:8080 回车
4. 如果出现apach界面，则安装成功
Linux 1.打开Terminal
2. 进入到 /Tomcat/bin 目录下，输入如下命令
$ ./startup.sh
3.如果出现Tomcat Started 类似的语句，则安装成功
Mac OS 执行/Library/Tomcat/bin下的startup.sh，然后打开 http://localhost:8080 查看是否Tomcat已经启动，若要停止服务器就运行同目录下的 shutdown.sh

MAVEN的安装及配置

  1. 点击MAVEN 进入列表，进入到对应版本的/binaries里面，找到合适的安装包

  2. 解压下载好的压缩包

  • 建议在安装时候，将文件夹命名为 Maven
系统 步骤
Windows 下载.zip文件解压
Linux 1. 将安装包安装到/usr/local 路径下
2. 解压压缩包
 · 如果格式为.tar.gz后缀 使用命令$ tar -zxvf
 · 如果格式为.rpm后缀 使用命令$ rpm -ivh
Mac OS 下载tar.gz文件 ，解压后放入 /Library 路径下
  1. 配置Maven环境
系统 步骤
Windows 1. 打开控制面板
2. 点击高级系统设置
3. 点击 高级 里面的环境变量
4. 在系统变量下面点击 新建 ,新建Maven环境变量
 · 变量名 M2_HOME
 · 变量值 Maven的目录(例如C:\Program Files\Maven)
5.双击系统变量下的Path变量
7. 新建字段：%M2_HOME%\bin
Linux 1. 修改环境变量(前提会用 vi 编辑器)
 · 终端输入$ vi /etc/profile
 · 在文件最后添加
  export M2_HOME=/usr/local/Maven
  export PATH=$PATH:$JAVA_HOME/bin:$CATALINA_HOME/bin:
   $M2_HOME/bin
2. 保存并退出vi
3. 终端输入$ source /etc/profile
Mac OS 1.打开Terminal 输入$ vi ~/.bash_profile
2. 添加下列两行代码:
  export M2_HOME=/Library/Maven
  export PATH=$PATH:$M2_HOME/bin
3. 保存并退出vi
4. 终端输入$source ~/.bash_profile
  1. 系统测试
系统 测试
Windows 1.打开CMD(win + R 输入CMD)
2. 输入$ mvn -version 回车
3.如果出现 Apache Maven 3.6.1等字样
Linux 1.打开Terminal
2. 输入$ mvn -version 回车
3.如果出现 Apache Maven 3.6.1等字样
Mac OS 1.打开终端
2. 输入$ mvn -version 回车
3.如果出现 Apache Maven 3.6.1等字样

总结
